> GNU:AEFR 目前并不是一个为了讨好所有人而设计的复杂应用，因为它仍未做出完整版本。它目前只是 1123 行直达底层的 Rust 逻辑所构成的一个内核。**在计算机的世界里，最短的路径永远是最无敌的！** 如果你追求的是目前的兼容性和易用，请去用 AA；如果你追求的是极致的自由与性能以及社区维护，欢迎来到 GNU:AEFR。

*   **非官方粉丝制作**: 一个使用纯 Rust 打造的，性能强劲的多平台、多线程《蔚蓝档案》二次创作编辑器。
*   **无游戏引擎**: 我们不依赖 Unity/Unreal，直接使用轻量级的 `egui` 库驱动图形界面。
*   **全平台制霸**: 原生支持 GNU/Linux、Android、macOS 和 Windows。

## 🚀 开始使用

> "Release? Real hackers compile from source." ;-)

前往 [**Releases**](https://github.com/OxidizedSchale/GNU-AEFR/releases) 页面下载源码，或直接获取为你的平台无指令集优化编译好的二进制文件。

GNU:AEFR 采用 **图形化** 与 **指令驱动 (Command-Driven)** 相结合的交互方式。

*   **桌面端**: 推荐使用图形化界面。
*   **移动端**: 在涉及文件导入时，目前只能使用指令。

点击界面左上角的 `[CMD]` 按钮即可打开内置调试控制台（有图形化支持）。

## 📖 指令参考手册

### 1. 场景与视觉
*   **加载背景**: `BG <图片路径>`
    *   瞬间切换背景图片，支持 `.jpg`, `.png`, `.webp`。
    *   示例: `BG C:\Assets\BlueArchive\BG_Classroom.png`
*   **装填角色**: `LOAD <槽位ID> <.atlas路径>`
    *   将角色加载到 `0` 到 `4` 号共 5 个槽位。加载成功后，控制台会打印出该角色可用的动作列表。
    *   示例: `LOAD 0 D:\Assets\Shiroko\Shiroko_Home.atlas`

### 2. 动作与演出
*   **切换动作**: `ANIM <槽位ID> <动作名> [loop: true/false]`
    *   `true` 为循环播放，`false` 为播放一次。
    *   示例: `ANIM 0 Start_Idle_01 true`

### 3. 剧本与对话
*   **发送对话**: `TALK <名字>|<学院>|<内容>`
    *   渲染标准的基沃托斯风格对话框，支持打字机效果。**必须使用竖线 `|` 分隔参数。**
    *   示例: `TALK 砂狼白子|对策委员会|老师，我们要去抢银行吗？`

### 4. 音频系统
*   **播放 BGM**: `BGM <音频路径>`
    *   异步加载并播放背景音乐，支持无缝切换。
*   **停止音乐**: `STOP`
    *   立即停止当前播放的 BGM。

---

### 💡 极客贴士
*   **路径问题**: Windows 推荐直接复制文件路径，AEFR 会自动处理引号；Android / Termux 请使用绝对路径。
*   **性能监控**: 得益于“绅士调度器”，即使填满了 5 个槽位，UI 线程依然保持丝滑。
*   **日志反馈**: 所有的操作结果都会实时打印在控制台的 Log 区域。

## 🤝 贡献指南
<details>
<summary><strong>点击展开：在参与构建 GNU:AEFR 时必须遵守的规则</strong></summary>

### 技术栈纯洁性
本项目坚持核心业务逻辑与架构 100% 使用 **Rust** 实现。
*   **原则上拒绝** 引入任何需要与 C++ 运行时或框架（如 Qt、Unity、Unreal 等）进行复杂交互的 PR。
*   **例外情况**: 允许为接入底层的 C 系统库编写 Rust 安全封装层，但必须优先使用成熟的 `-sys` 绑定。

### `unsafe` 准则
“非必要不使用，使用时必须万无一失”。任何 `unsafe` 块必须在注释中提供“必要性证明”。
</details>

## 🏛️ 架构哲学
<details>
<summary><strong>点击展开：了解驱动 GNU:AEFR 的硬核底层设计</strong></summary>

### “绅士调度器”：算力层面的阶级隔离
我们不信任 OS 默认调度。调度器会自动探测核心数 `N`，并隔离出核心用于特定任务：
*   **1 核心** 预留给 UI/渲染（主线程）。
*   **1 核心** 预留给 OS/音频，杜绝爆音。
*   **其余核心** 分配给 Worker 线程池进行 Spine 计算。

### GUI 规则
UI 组件必须遵循 `egui` 的**即时模式 (Immediate Mode)** 哲学。
</details>
